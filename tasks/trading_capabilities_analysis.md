### 2. 已实现的交易策略

#### 策略一：均线交叉策略 (MA Cross)
- **类型**: 趋势跟随
- **逻辑**: 快速均线（默认20期）上穿慢速均线（默认50期）做多，下穿做空
- **信号强度**: 0-1 连续值
- **适用场景**: 趋势行情

#### 策略二：突破策略 (Breakout)
- **类型**: 动量/趋势跟随
- **逻辑**: 收盘价突破近期高点做多，跌破近期低点做空
- **回望周期**: 50 根 K 线（可配置）
- **适用场景**: 震荡后的趋势启动

#### 策略三：均值回归策略 (Mean Reversion)
- **类型**: 均值回归
- **逻辑**: 当收盘价低于开盘价时做多（预期回归），反之做空
- **置信度**: 0.6（低于趋势策略）
- **适用场景**: 震荡行情

#### 策略四：Z-Score 统计策略
- **类型**: 统计套利/均值回归
- **逻辑**: Z 分数超过阈值做空（过度偏离，预期回归），低于负阈值做多
- **置信度**: 0.8
- **适用场景**: 价格偏离均值的统计套利

#### 策略五：加权集成策略 (Weighted Ensemble)
- **类型**: 多策略融合
- **逻辑**: 将多个信号按权重加权求和，综合打分
- **公式**: `score = Σ(model_score × weight)`
- **适用场景**: 通过多策略互补提高胜率

### 5. 风控体系

系统拥有完整的多层风控架构：

#### 规则一：杠杆上限 (Leverage Cap)
- 默认最大杠杆 3x（可按标的覆盖）
- 超限时自动缩减仓位或拒绝

#### 规则二：最大回撤 (Max Drawdown)
- 默认阈值 20%
- 触发后进入 KILL 状态或 REJECT
- 允许减仓单通过（防止"死亡螺旋"）

#### 规则三：最大持仓 (Max Position)
- 限制单标的最大持仓量和名义价值
- 支持自动缩减建议

#### 熔断开关 (Kill Switch)
- **5 级作用域**: 全局 → 账户 → 组合 → 策略 → 标的
- **2 种模式**: 硬熔断（拒绝一切）/ 仅减仓（只允许平仓）
- **TTL 自动过期**: 设定时长后自动恢复
- **完整审计**: 触发来源、原因、时间全部记录

#### 压力测试
- 场景分析框架（价格冲击、波动率飙升）
- 评估组合在极端行情下的损失

### 6. 执行基础设施

| 组件 | 说明 |
|------|------|
| **幂等性** | 基于 idempotency_key 去重，防止重复下单 |
| **重试策略** | 最多 3 次，指数退避（0.1s → 2s） |
| **限速器** | 令牌桶算法，10 请求/秒 |
| **熔断器** | 8 次失败触发，5 秒冷却期 |
| **对账** | 订单/成交一致性校验 |

---

### 7. 仓位管理

**固定比例仓位管理器 (Fixed Fraction Sizer)**:
```
target_qty = (权益 × 风险比例 × |权重|) / 价格
```
- 默认每笔风险 2% 权益
- 支持按 lot_size 取整（适配合约最小单位）

**分配器**:
- **等权分配**: 在 N 个候选标的间平均分配
- **单资产分配**: 最强信号获得全部仓位

## 三、开发完成后可以做什么交易（总结）

### 立即可用

1. **Binance 永续合约的趋势跟随交易**
   - 使用 MA Cross 或 Breakout 策略
   - 支持 BTC、ETH 等所有 USDT-M 永续合约
   - 多空双向交易，杠杆可调

2. **均值回归交易**
   - 使用 Mean Reversion 或 Z-Score 策略
   - 捕捉价格偏离均值后的回归机会

3. **多策略集成交易**
   - 通过 Weighted Ensemble 融合多个信号
   - 趋势策略 + 均值回归策略互补

4. **历史回测与策略验证**
   - 基于 CSV 历史数据完整回测
   - 输出收益率、夏普比率、最大回撤等统计

5. **带完整风控的实盘交易**
   - 杠杆限制、回撤保护、持仓上限
   - 多级熔断机制保障资金安全

### 需要扩展才能做的交易

|----------|-----------|------|

| **网格交易** | 新增网格策略信号模型 | 较小 |

## 四、上线前需要解决的问题

### 紧急 (Critical)

1. **API 密钥安全** - 当前通过 `os.getenv()` 读取，需确保不硬编码
2. **RiskAggregator 竞态条件** - 统计更新在锁外，多线程下数据可能损坏
3. **EventDispatcher 内存泄漏** - `_seen_event_ids` 集合无限增长

### 重要 (High)

4. **测试覆盖率 < 5%** - 核心交易逻辑需要全面测试
5. **SimVenue 未完成** - 模拟交易缺少延迟和滑点建模
6. **约 55% 的文件为空壳** - Attribution、Policy 等模块未实现
7. **Decimal/float 混用** - 回测引擎中存在精度风险

## 五、架构优势

- **事件驱动** - 松耦合，易扩展
- **不可变状态** - 冻结 dataclass，确定性重放
- **协议接口** - 可插拔的策略、风控、分配器
- **零外部依赖** - 核心交易逻辑仅用标准库
- **机构级风控** - 多层级熔断、审计追踪、压力测试
- **幂等执行** - 去重缓存防止重复下单
